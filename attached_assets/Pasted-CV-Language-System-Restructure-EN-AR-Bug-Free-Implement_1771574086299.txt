→ Global header toggle must automatically switch to the same language.

If user changes language from Global Header,
→ CV Edit + Preview must update instantly.

Everything must stay synchronized.

2️⃣ Content Behavior

When switching language:

All descriptions in edit fields must show content from that selected language.

Preview page must load the same selected language content.

No mixing of English and Arabic.

No reload bugs.

No partial updates.

🧠 New Required Architecture (Important)

To avoid complexity and timing issues, we will change the database structure.

🔹 Rule:

Every AI-generated content MUST generate in BOTH languages at the same time.

When content is generated:

Generate English version

Generate Arabic version

Store both properly in database

🗄️ Database Structure (Clean Approach)
For Guest Users:

Create TWO separate tables:

guest_user_cv_en

guest_user_cv_ar

Both tables:

Match records using email as primary key

Store same CV sections in respective languages

When toggle switches:

If EN → fetch from guest_user_cv_en

If AR → fetch from guest_user_cv_ar

For Registered Users:

Current:

Existing cv table → use ONLY for English

Create new table: cv_ar → store Arabic version

When user registers:

If guest data exists with same email:

Migrate BOTH English + Arabic data

Save into:

cv (English)

cv_ar (Arabic)

Do NOT delete one language accidentally.

🔁 Content Flow Summary

User generates CV

System generates:

English version

Arabic version

Store both in respective tables

Toggle only controls:

Which table to fetch data from

Edit + Preview always read from selected language table

⚠️ Critical Requirements

No delay issues

No content mixing

No data loss during guest → registered migration
