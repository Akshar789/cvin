Profile Settings – Full Education & Work Experience Sync (Repeater Support Required)
🎯 Objective

We must properly align Profile Settings with the CV step flow — especially:

Education (repeater)

Work Experience (repeater)

Currently, this is not fully aligned.

We need:

All step fields saved correctly in database

All repeater entries stored properly

All data displayed correctly in Profile Settings

No data loss

No missing entries

Clean mapping

Future-safe for second CV generation

⚠️ Critical Requirement

Education and Work Experience BOTH support repeater fields.

Profile Settings must also support repeater display.

It must NOT show only a single entry.

It must show ALL entries saved in database.

🧩 1️⃣ Education – Required Fields (From Steps)

We collect:

Institution

Degree

Field of Study

Start Year

Graduation Year

Description (optional)

Required Fix:
A. Database

Ensure every education entry is saved properly

Each entry must be linked to:

CV ID

User ID

Must support multiple entries

No overwriting of previous entries

B. Profile Settings Display

In Settings → Education section:

Show ALL saved education entries

Use repeater-style layout

Display:

Institution

Degree

Field of Study

Duration (Start Year – Graduation Year)

Description (if exists)

If Description is empty:
→ Show education entry without description
→ Do NOT hide entire entry

🧩 2️⃣ Work Experience – Required Fields (From Steps)

We collect:

Company Name

Job Title

City

Start Date (Month + Year)

End Date (Month + Year OR Present)

Description (optional)

Supports repeater.

Required Fix:
A. Database

Ensure every work experience entry is saved properly

Must not overwrite previous jobs

Must be linked to correct CV

Must store:

Start Month

Start Year

End Month

End Year

Present flag (if selected)

B. Profile Settings Display

In Settings → Work Experience:

Display ALL saved positions

Properly formatted:

Job Title at Company

City

Duration (Start – End or Present)

Description (if exists)

If Description is empty:
→ Still show job entry.
