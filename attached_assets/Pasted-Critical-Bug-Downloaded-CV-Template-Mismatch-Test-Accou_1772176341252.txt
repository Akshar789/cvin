Preview and Download engines are not aligned.

🔎 Root Cause Investigation Required

We must check:

1️⃣ Is Download Using Correct Template ID?

Verify during download:

What cv_id is being used?

What template_id is being passed?

Is it reading from:

Database?

Cached state?

Default template?

Log before generating PDF:

CV ID

Template ID

Template Name

They MUST match the preview template.

2️⃣ Check If Download Uses Default Template

Possible issue:

Download engine may be doing something like:

template = templates[cv.template_id] || defaultTemplate

If template_id not updated before signup redirect,
it may fallback to default template.

Remove any fallback behavior.

3️⃣ Check Signup Redirect Flow

Important:

When user signs up mid-flow:

Is CV saved correctly before auth?

Is template_id preserved?

Is new CV created?

Is template_id overwritten?

Likely problem:

During signup,
CV state may be re-initialized,
causing template mismatch.

4️⃣ Check Timing Issue

Possibility:

Template selection updated in UI

But not persisted to database yet

Download triggered before DB update completes

Fix:

Before download:

Ensure latest template_id is persisted.

Await save completion.

Then generate file.

5️⃣ Ensure Preview & Download Use Same Rendering Engine

Preview looks correct.
Download looks wrong.

This means:

They use different rendering logic.

Or different template mapping.

Or download uses outdated template mapping.

Fix:

Preview and Download must use:

Same template renderer
Same template_id
Same theme variables
Same layout config
