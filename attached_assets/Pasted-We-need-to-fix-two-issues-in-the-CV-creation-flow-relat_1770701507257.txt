🛠️ Issue 1: Work Experience Description Not Showing in Resume Preview (AI CVs)
❌ Current Problem

When a CV is generated using AI Generation:

In the editor, Work Experience Description fields are present and editable.

But in the resume preview, these descriptions are not displayed.

This issue does NOT occur when the CV is created using Manual Continue.

✅ Expected Fix

Ensure that Work Experience description data generated by AI:

Is properly saved (state / DB).

Is passed to the resume preview component.

Resume preview should:

Display description content for each work experience entry.

Behave the same way as it does for manually created CVs.

Maintain consistency between:

Editor view

Resume preview view

🛠️ Issue 2: Incorrect Loader Display on Buttons
❌ Current Problem

After filling Basic Info:

When the user clicks Manual Continue:

The AI Generation button loader also appears.

This misleads users into thinking AI is running.

✅ Expected Fix

Button loaders must be action-specific:

Clicking AI Generation → show loader only on AI Generation button

Clicking Manual Continue → show loader only on Manual Continue button

Ensure:

No shared loading state between both buttons.

Each button has its own loading flag/state.
