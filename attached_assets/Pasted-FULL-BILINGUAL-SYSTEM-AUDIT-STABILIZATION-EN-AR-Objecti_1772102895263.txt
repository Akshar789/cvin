🧠 PHASE 1 — DEFINE A SINGLE SOURCE OF TRUTH
1️⃣ Language State Authority

There must be ONE authoritative language source.

Example:

Global app state (Redux / Context / i18n)

NOT local state per component

NOT URL + localStorage mixed

NOT duplicate states

Every function must read from this single source.

Log:

Current language at execution time

🗄 PHASE 2 — DATABASE STRUCTURE VALIDATION

Every bilingual field must follow strict structure:

   en: "...",
   ar: "..."
}

education[].description

target_job_domain

location

AI-generated content

No flat strings allowed.

No partial bilingual objects allowed.

🔁 PHASE 3 — SAVE LOGIC STANDARDIZATION
RULE:

Whenever a field is saved:

If currentLanguage === "en":

Save value to field.en

Auto-generate field.ar

Save both

If currentLanguage === "ar":

Save value to field.ar

Auto-generate field.en

Save both

No fallback.
No OR logic.
No merging.

Never use:

field.en || field.ar

Never use:

field[currentLanguage] || field.en

That causes language bleed.

🔄 PHASE 4 — RENDER LOGIC STANDARDIZATION

When rendering:

If language = EN → display .en
If language = AR → display .ar

If selected language content is empty:
→ show empty
→ do NOT fallback automatically

Fallback causes mixing.
